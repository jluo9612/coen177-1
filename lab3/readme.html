Compare performance of disk reading between reading sequentially and jumping all over the file.

Files

  • seqread.c: Program for reading sequentially.
  • randread.c: Program for reading randomly (jumping all over the file). Also check for the file size and read roughly the same amount of data as seqread.
  • makefile: For compiling.

Usage

After compiling the programs into executables, run the executable and pass the path to the file to read as a command line argument.

Ex: ./seqread /local/weka/random5G

Analysis

Within the same method of reading (sequential and random), the time it takes to read the file is roughly proportional to the file's size. Also, reading randomly takes ~94-108 times longer than the sequential case. Clearly, it is a very bad idea to change the file cursor multiple times while reading. Note that the I ran randread on 5G file at about 10 pm separate from other experiments, so the performance seems to be a bit better than other cases.
